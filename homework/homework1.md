# PolyStudio 长视频 Agent 基础能力分析

## 1. 现有工作流

```
用户输入 → 分镜生成（等待确认）→ 生图（按分镜）→ 图生视频 → 拼接成片
```

**步骤分解**：

| 步骤     | 工具                                                                         | 数据传递                                |
| -------- | ---------------------------------------------------------------------------- | --------------------------------------- |
| 分镜生成 | Agent（LLM 生成）                                                            | 用户需求 → 分镜脚本（含镜头描述、时长） |
| 生图     | `generate_volcano_image_tool`（空镜）<br>`edit_volcano_image_tool`（角色镜） | 分镜脚本 → 图片路径                     |
| 图生视频 | `generate_volcano_video_tool`（mode="image"）                                | 图片路径 + 分镜描述 → 视频片段路径      |
| 拼接     | `concatenate_videos_tool`                                                    | 视频片段列表 → 最终视频                 |

数据流：`分镜脚本（文本）` → `分镜图片（路径）` → `视频片段（路径）` → `最终视频（路径）`。


## 2. 现有角色一致性

**核心机制**：依赖已有角色图 + `edit_volcano_image_tool` 而非 `generate_volcano_image_tool`。

| 场景                    | 工具选择                         | 规则依据                                                         |
| ----------------------- | -------------------------------- | ---------------------------------------------------------------- |
| 有角色镜头 + 已有角色图 | `edit_volcano_image_tool`        | Prompt 第 140-144 行：有角色镜头**必须**用 edit，禁止用 generate |
| 有角色镜头 + 无角色图   | Agent 暂停，提示用户先生成角色图 | Prompt 第 136-138 行                                             |
| 空镜/场景镜头           | `generate_volcano_image_tool`    | Prompt 第 141 行                                                 |

**本质**：角色图作为"锚点"，通过 edit 在保持身份特征的同时生成新场景。API 层面使用 Seedream 的图像编辑能力实现一致性。

## 3. 现有时长处理

**时长来源**：分镜脚本中指定（如"镜头1：5秒"），火山引擎 API 支持 4-12 秒/片段（默认 5 秒）。

**拼接后校验**：

- `concatenate_videos_tool` 内部记录总时长（`final_clip.duration`），但**代码中未见对期望时长与实际总时长的差值校正逻辑**.
- 实际返回结果仅含拼接后视频路径，无独立时长校验步骤.

**接口返回时长不一致处理**：

- `generate_volcano_video_tool` 的 `duration` 参数为请求参数，非实际返回值。
- 实际生成视频时长由 API 决定，工具层未做"请求 vs 实际"的对齐处理。
- 拼接层也无重分配或告警机制。

## 总结

- **单角色一致性**：依赖"角色图锚点 + edit 工具"，规则明确。
- **固定时长**：分镜预设时长，拼接无动态校正——接口层"请求时长"与"实际时长"之间存在 gap。
- **扩展方向提示**：多角色需解决多锚点图管理；动态时长需在拼接层增加校验/重分配逻辑。
