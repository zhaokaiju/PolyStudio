# 基于 Skill 的自动化多模态内容二创方案

> 本文档设计一个一键式自动化二创工作流 Skill，利用 PolyStudio 的 Skill 扩展机制与多模态能力，实现从原始视频到二创内容的全自动化生成。

## 一、背景与目标

### 1.1 现有能力

- **多模态理解**：通过 `qwen_omni_understand` 工具支持图片 / 音频 / 视频智能分析。
- **多模态生成**：支持智能播客（文字→TTS→混音）、虚拟人视频（图片+音频→唇形同步）、文生图/视频/3D。
- **Skill 扩展系统**：通过编写 `SKILL.md` 文件为 Agent 注入特定领域专业知识，支持渐进式加载。

### 1.2 核心目标

用户上传一段原始视频并触发该 Skill 后，Agent 自动完成：

```
原始视频 → 内容理解 → 脚本生成 → 二创内容生产 → 最终交付
```

整个过程**无需用户中途干预**，实现"一键式"体验。

```markdown
# 自动化多模态内容二创 Skill

## 概述

本 Skill 实现从原始视频到二创内容的全自动化流水线。用户只需上传视频并触发本 Skill，Agent 将自动完成全部处理流程。

### 支持的输出模式

| 模式             | 说明     | 输出            |
| ---------------- | -------- | --------------- |
| `podcast`        | 播客风格 | MP3 音频 + 字幕 |
| `news_broadcast` | 新闻播报 | 虚拟人视频      |
| `commentary`     | 解说评论 | 虚拟人视频      |
| `summary`        | 内容摘要 | 文字稿          |

## 核心工作流程

### Step 1：多模态内容理解

**工具调用**：`qwen_omni_understand`

**处理逻辑**：

1. 提取视频的核心主题与叙事结构
2. 识别关键人物、场景、事件
3. 提取核心观点与情感基调
4. 生成时间轴标记（关键片段 timestamp）

**Prompt 模板**：
请分析这段视频的完整内容：

1. 视频的主题是什么？核心信息是什么？
2. 视频的结构是怎样的？（开场/发展/高潮/结尾）
3. 关键人物有哪些？他们的主要观点或行为是什么？
4. 视频的情感基调是什么？
5. 如果要改编成 {target_mode}，需要保留哪些核心要素？
   请以结构化方式输出分析结果。

**输出数据结构**：
{
"theme": "...",
"core_message": "...",
"structure": { "opening": "...", "development": "...", "climax": "...", "ending": "..." },
"key_characters": [...],
"emotional_tone": "...",
"timestamp_markers": [{ "time": "00:01:23", "description": "..." }],
"retainable_elements": [...],
"adaptation_notes": "..."
}

### Step 2：脚本生成

**工具调用**：LLM 内部处理（无外部工具调用）

**处理逻辑**：

1. 根据 `target_mode` 选择对应脚本模板
2. 将 Step 1 的分析结果注入模板
3. 生成完整脚本（包含转场提示、时间标记）

**输出格式示例（podcast 模式）**：

# 播客脚本

## 元信息

- 原始视频主题：{theme}
- 风格：{emotional_tone}
- 预计时长：{estimated_duration}

## 正文

[开场 - 00:00~00:30]
欢迎收听今天的播客节目...

[主体内容 - 00:30~05:00]
首先，让我们来看看这个视频的核心观点...

[结尾 - 05:00~05:30]
以上就是今天的主要内容...

### Step 3：多模态内容生产

根据 `target_mode` 调用对应生成工具：

#### 模式 A：播客生成（podcast）

**工具调用链**：

1. qwen_tts（qwen_voice_cloning 或 qwen_voice_design）
   └─ 输入：脚本文字
   └─ 输出：人声音频

2. select_bgm_tool
   └─ 输入：情感基调
   └─ 输出：BGM 文件

3. mix_audio_with_bgm_tool
   └─ 输入：人声音频 + BGM
   └─ 输出：最终播客 MP3

**Voice 选择策略**：

- 根据 `emotional_tone` 自动选择或克隆合适音色
- 紧张/严肃 → 专业低沉音色
- 轻松/活泼 → 明快年轻音色

#### 模式 B：虚拟人视频（news_broadcast / commentary）

**工具调用链**：

1. qwen_tts（生成配音）
   └─ 输入：脚本
   └─ 输出：音频

2. detect_face_tool（如果用户未提供形象）
   └─ 输入：视频中人物
   └─ 输出：人脸图片

3. generate_virtual_anchor_tool
   └─ 输入：人脸图片 + 音频
   └─ 输出：唇形同步视频

**虚拟人形象策略**：

- 优先级：用户指定形象 > 视频中检测到的人物 > 默认形象
- 新闻播报模式：正装、端庄
- 评论解说模式：可更活泼
```

## 六、关键技术实现细节

### 6.1 Skill 激活判断

Agent 通过 `description` 字段匹配用户意图：

```yaml
description: |
  自动化多模态内容二创专家。当用户上传视频并要求"二创"、"改编"、
  "生成播客"、"生成虚拟人视频"时自动激活。
```

匹配关键词：`视频` + (`二创` OR `改编` OR `生成播客` OR `生成虚拟人` OR `自动化`)

### 6.2 内部状态管理

Skill 执行过程中使用**临时文件**存储中间结果：

```python
# 内部状态路径
/workspace/remix_state.json

{
  "video_path": "/storage/videos/xxx.mp4",
  "analysis_result": { ... },
  "script": "...",
  "target_mode": "podcast",
  "intermediate_files": {
    "tts_audio": "/storage/audios/tts_xxx.wav",
    "bgm": "/storage/bgm/selected_xxx.mp3",
    "final_output": "/storage/podcasts/output_xxx.mp3"
  }
}
```

### 6.3 错误处理与回退

| 场景               | 处理策略                       |
| ------------------ | ------------------------------ |
| 视频理解失败       | 返回错误，提示用户检查视频格式 |
| TTS 生成失败       | 降级为纯文字输出               |
| 虚拟人形象检测失败 | 使用默认形象                   |
| 用户中断           | 保留中间结果，支持续传         |

### 6.4 多模态工具调用顺序

```
qwen_omni_understand (视频分析)
         │
         ▼
    分析结果
         │
         ▼
    LLM 生成脚本（内部处理）
         │
    ┌────┴────┐
    ▼         ▼
 podcast    virtual_anchor
    │         │
    ▼         ▼
 qwen_tts   qwen_tts
    │         │
    ▼         ▼
 select_bgm +───▶ detect_face
    │                 │
    ▼                 ▼
 mix_audio  generate_virtual_anchor
    │         │
    └────┬────┘
         ▼
    最终输出
```

- **教程类视频** → 解说评论模式，增加补充说明。

## 十、总结

本方案通过 PolyStudio 的 Skill 扩展机制，将多模态理解工具（`qwen_omni_understand`）与多模态生成工具（TTS、混音、虚拟人）串联成一个自动化流水线，实现：

1. **一键触发**：用户只需上传视频并说明意图。
2. **全自动执行**：视频理解 → 脚本生成 → 内容生产全程无需干预。
3. **多模式输出**：支持播客、虚拟人视频等多种输出形态。
4. **渐进式加载**：Skill 内容按需加载，不占用初始上下文。
5. **状态可视化**：通过 SSE 事件实时展示处理进度。

该 Skill 可作为 PolyStudio 多模态工作流定制的标杆案例，验证 Skill 机制对复杂自动化场景的支撑能力。
